  Evidence:

      1. data.gov.in API

         Endpoint: POST /api/schemes/gov/pmjay/check
         URL: https://api.pmjay.gov.in/eligibility/check
         Input: Aadhaar + family size → eligibility check

      5. National Scholarship Portal
         Endpoint: GET /api/schemes/gov/scholarship
         URL: https://api.scholarships.gov.in/api/schemes
         Params: category, income_limit, education_level

  - Unified Search:
      GET /api/schemes/gov/search-all?q=...
      Searches across all 5 APIs simultaneously

  - AI Scheme Matching (POST /api/schemes/discover):
      * Demographic matching engine
      * Scores schemes against citizen profile
      * Sarvam AI text analytics for classification

  - Sarvam AI APIs (server/routes/sarvam.js):
      * POST /api/sarvam/translate

────────────────────────────────────────────────────────────────────────────────
Req 4.1: Security Best Practices
         "Implements authentication, data protection, session management"
────────────────────────────────────────────────────────────────────────────────

  Verdict: [MET]

  Evidence:
  - Authentication (server/routes/auth.js + authMiddleware.js):
      * Aadhaar lookup → OTP generation → OTP verification → JWT
      * JWT tokens: 2-hour expiry, signed with JWT_SECRET
      * 3 middleware tiers: verifyToken, optionalAuth, requireAdmin
      * Admin: separate bcrypt-hashed credentials

  - Session Management:
      * sessionSecurity (security.js):
        - Browser fingerprinting (userAgent + language + timezone +
          screen resolution + color depth + platform)
        - Session creation with fingerprint binding
        - Session validation: checks fingerprint match + 2h expiry
        - clearSession() on logout/timeout
      * InactivityHandler: auto-logout after 5 min idle
      * Session data in sessionStorage (cleared on tab close)

  - Data Protection:
      * XSS prevention: sanitizeInput() strips < > " ' & / \
      * sanitizeFormData() recursively sanitizes all string fields
      * CSRF tokens: server issues, client sends in X-CSRF-Token header
      * CSP meta tag: restrictive Content-Security-Policy
      * File upload validation: type whitelist, 5MB limit, magic bytes

  - Rate Limiting (server/index.js):
      * Global: 100 requests / 15 minutes per IP
      * Auth routes: 10 requests / 15 minutes per IP

  - CORS: Configured with specific origin whitelist

  - docs/reports/THREAT_MODEL.md (200 lines):
      * STRIDE analysis with 20 enumerated threats
      * 7 asset categories identified
      * 12 security controls documented
      * Residual risks + planned mitigations

  Gap:
      * Passwords in .env could use vault
      * No HTTPS enforcement (relies on Vercel)
      * OTP is simulated (hardcoded "123456" in dev)
